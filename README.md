# 🧭 Workflow حرفه‌ای Git (Commit + Branching)

## 🔱 ساختار برنچ‌ها

- `main` → نسخه پایدار (Production)
- `develop` → محل توسعه اصلی
- `feature/*` → افزودن قابلیت جدید
- `fix/*` → رفع باگ

مثال:

```
feature/login-form
fix/form-validation
```

---

## ✍️ Conventional Commits (الگوی کامیت)

فرمت استاندارد:

```
<type>(scope): message
```

### Typeهای پرکاربرد:

- `feat` → ویژگی جدید
- `fix` → رفع باگ
- `style` → فقط تغییر ظاهری (CSS، فاصله، فرمت)
- `refactor` → بازنویسی بدون تغییر رفتار
- `docs` → مستندات
- `chore` → تنظیمات، کانفیگ

### مثال‌های حرفه‌ای:

```
feat(auth): add login form UI
fix(form): prevent empty submit
style(header): improve mobile layout
refactor(api): simplify fetch logic
```

---

## 🔄 روند کار روزمره

1️⃣ ساخت برنچ از develop

```
git checkout develop
git checkout -b feature/home-page
```

2️⃣ کدنویسی + کامیت‌های کوچک و معنادار

3️⃣ مرج به develop

```
git checkout develop
git merge feature/home-page
```

4️⃣ وقتی پروژه آماده شد → مرج develop به main
